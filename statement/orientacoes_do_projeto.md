A nota prática da disciplina corresponde ao desenvolvimento de um software a ser produzido ao longo de todo o semestre letivo. A seguir, as orientações para o desenvolvimento do trabalho:

O trabalho deverá ser realizado em grupo de até 3 alunos.
O projeto consistirá no desenvolvimento de um software, que:
deve apresentar um nível mínimo de complexidade. Aplicativos que compreendam apenas cadastros e relatórios não serão admitidos. Algumas métricas para avaliar o tamanho do projeto: aprox. 12 requisitos funcionais , aprox. 5 regras de negócio com alguma complexidade, aprox. 6 classes de domínio.
sistemas “batidos” não serão aceitos, como por exemplo, sistema de pedidos, controle de estoque, locadora,...
considere que o sistema deve funcionar ao final do semestre e que não está no escopo desta disciplina linguagem de programação e tecnologias específicas. Desta forma, comprometam-se com projetos que o grupo domine a tecnologia necessária, por exemplo, se não dominam banco de dados, pensem em solução que possa ser implementada apenas com serialização.
A modelagem UML deverá usar a ferramenta Enterprise Architect:
Possível de uso em máquina própria a partir do seguinte procedimento:
Acesse o compartilhamento \\pastas.ufsc.br\Instaladores\EnterpriseArchitect\
Se tiver dificuldade em acessar veja o tutorial em https://otrs.setic.ufsc.br/otrs/public.pl?Action=PublicFAQZoom;ItemID=460. Você deve estar na rede UFSC para acesso.
Executar o arquivo Instalar Enterprise Architect.lnk
Se encontrar problema através deste link, faça o procedimento manualmente:
Instale: \\pastas.ufsc.br\instaladores\EnterpriseArchitect\Files\SetupFull.msi
Registre a licença: \\pastas.ufsc.br\instaladores\EnterpriseArchitect\License\KeyStore.reg
Atenção: esta licença funciona apenas na rede UFSC. Acessos externos devem ocorrer através de VPN. Dúvidas sobre a VPN UFSC (http://setic.ufsc.br/servicos/acesso-a-redeufsc/servico-de-vpn-virtual-private-network/)
Caso solicite o servidor de licenças (Keystore Server), é "ssks://licenciador2012.setic.ufsc.br"
Após instalação, abre o aplicativo e entre com o código de ativação 8BSX. Clique em Aceitar (I Accept) e em Ok
A implementação deverá ser em Python ou Java
O conteúdo a ser entregue (em um arquivo .ZIP) consiste em:
Especificação de requisitos, conforme template tratado na disciplina
Especificação de projeto em UML, desenvolvida no software Enterprise Architect (entregar o arquivo .eap). Conteúdo mínimo:
Visão geral do projeto, requisitos e rastreabilidade
Diagrama(s) de classes
Diagrama(s) de casos de uso
Diagramas de atividades para refinamento dos casos de uso
Diagramas de sequência para refinamento dos casos de uso
Código que execute sem erros;
Notas de entrega (release notes): texto com instruções de instalação / manutenção para futuros usuários do software; explicitação dos aspectos positivos e negativos da versão produzida (lembrando que é natural que a primeira versão não seja a melhor possível); eventuais restrições contidas na versão entregue.
A entrega do trabalho será gradual e seguirá o calendário divulgado no Moodle.
Trabalhos que não apresentarem o programa em funcionamento no momento da apresentação ao professor, não estarão aptos a apresentação.
